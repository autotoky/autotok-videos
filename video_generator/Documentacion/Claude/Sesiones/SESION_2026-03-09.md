## Trabajo realizado

### 1. Investigacion TikTok data (QUA-36)

Investigacion de como obtener datos de engagement y ventas de TikTok:
- **Engagement**: disponible via paginas publicas (`tiktok.com/@cuenta/video/ID`), JSON embebido `__UNIVERSAL_DATA_FOR_REHYDRATION__` contiene todas las metricas
- **Ventas**: solo disponibles en la app movil de TikTok, NO en navegador
- `affiliate.tiktok.com` requiere cuenta seller (Sara es afiliada)
- `creator.tiktok.com` redirige a TikTok Studio

Sara sugirio el enfoque de scrapear la pagina publica del video (visible en el lateral de cada publicacion), que resulto ser el mas sencillo y efectivo.

### 2. Tablas BD (Turso)

Creadas dos nuevas tablas:
- **`video_stats`**: engagement + ventas por video (upsert por video_id), campos: views, likes, comments, shares, saves, units_sold, gmv, commission
- **`video_stats_history`**: historial temporal para tracking de cambios

### 3. Stats Scraper (stats_scraper.py)

Script Python que:
- Lee videos publicados (con tiktok_post_id) de la BD
- Visita la pagina publica de cada video en TikTok
- Extrae metricas del JSON embebido (zero deps — solo urllib)
- Guarda en video_stats + video_stats_history
- Rate limiting: 2-4s aleatorio entre peticiones
- Maneja videos no publicos (status_self_see, status_deleted)

Primer run: 54/90 OK (36 privados/eliminados), 40,539 views totales.

### 4. Dashboard de Estadisticas (/api/stats)

Nuevo endpoint en Vercel con dashboard interactivo:
- **KPIs**: views, likes, comments, shares, saves, videos, avg views, GMV
- **Pestana "Por Producto"**: tabla con metricas agregadas por producto
- **Pestana "Por Deal Math"**: tabla con metricas por tipo de deal math + engagement rate
- **Pestana "Todos los Videos"**: lista completa con link a TikTok
- **Pestana "Ventas Manual"**: formulario para registrar ventas/GMV/comision por video
- Filtros por cuenta y producto
- PIN compartido con dashboard de estado (misma cookie)
- Link de navegacion cruzada desde dashboard de estado

### 5. Import historico TikTok Studio (QUA-183)

Herramientas para capturar videos desde TikTok Studio (virtual scrolling) y asignar `tiktok_post_id` a videos en BD:

#### scroll_capture_tiktok.js (v2.0)
Script de consola para auto-scroll en TikTok Studio > Publicaciones:
- v1.0: captura basica con scroll del document
- v2.0: busca contenedor scrollable, detecta el MAS GRANDE (no el mas pequeno — bug critico en primera version)
- Minimo clientHeight 200px para evitar divs pequenos
- Espera lazy load, 8 reintentos al llegar al fondo
- Output: `1. POST_ID | FECHA | VIEWS views | TITULO`

**Bug critico en v2.0 inicial**: `findScrollContainer()` buscaba el div mas PEQUENO (`bestHeight = Infinity`, comparaba `< bestHeight`). Encontraba un div de 36px en vez del contenedor real. Fix: cambiar a buscar el mas GRANDE (`bestHeight = 0`, comparar `> bestHeight`, minimo 200px).

#### import_studio_html.py (v4.0)
Parser de output del script de consola + HTML directo:
- `is_console_format()` / `parse_console_output()` — detecta y parsea formato de consola
- `parse_file()` — auto-detector de formato (consola vs HTML)
- `parse_studio_date()` — soporta fechas con ano ("5 dic 2025, 22:30") y sin ano ("9 mar, 14:30")
- CLI: `python import_studio_html.py archivo.txt --cuenta X [--dry-run]`

### 6. Publicacion trendy dia 10 desde PC operadora (QUA-184)

Sara quiso publicar 23 videos de ofertastrendy20 para el 10 de marzo desde el PC de Carol (Sara-Yeast). Se encontraron 4 problemas:

#### Problema 1: config_operadora.json compartido via Synology
`config_operadora.json` en `kevin/` se sincroniza a todos los PCs. Estaba configurado para totokydeals + ruta de Mar del dia anterior.
**Fix**: cambio manual a ofertastrendy20 + ruta Sara-Yeast.

#### Problema 2: Chrome no encontrado (WinError 2)
Chrome en PC de Carol esta en `Program Files (x86)`, no en `Program Files`.
**Fix**: nueva funcion `_find_chrome()` en `tiktok_publisher.py` que auto-detecta Chrome en multiples rutas. `get_cuenta_config()` verifica que chrome_path de config existe, si no cae a `_find_chrome()`.

#### Problema 3: "No hay videos pendientes"
`publicar_facil.py` llama API primero; si la API responde (incluso sin el lote esperado), no busca archivos locales.
**Workaround**: usar `tiktok_publisher.py --lote RUTA` directamente.

#### Problema 4: filepath apunta a C:/Users/gasco/...
Los lotes guardan rutas del PC de Sara. En Synology los videos estan planos. La API sobreescribe ediciones manuales al JSON local (linea 106 de publicar_facil.py).
**Fix**: fallback por filename en `tiktok_publisher.py` — extrae `os.path.basename()` y busca en `drive_path/cuenta/filename.mp4`.

**Resultado final**: publicacion exitosa de los 23 videos tras aplicar los 3 fixes.
