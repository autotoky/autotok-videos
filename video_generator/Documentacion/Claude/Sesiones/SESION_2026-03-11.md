## Parte 10: Fixes generador — Pipeline end-to-end con formato_material

### Contexto
Al intentar generar videos de formatos nuevos (VIVONU shilajit #32/#33), el pipeline completo fallo en 4 puntos distintos. Cada fix se hizo iterativamente con Sara testeando entre cada uno.

### Fix 1 — CLI mostraba 0 audios
El CLI contaba audios desde `audios WHERE bof_id = X` (sistema viejo). Con formato_material, los audios se asocian por la tabla intermedia.
- **Fix:** Query cambiada a `(SELECT COUNT(*) FROM formato_material WHERE bof_id = pb.id AND tipo = 'audio')` en 2 ubicaciones (~linea 782 y ~linea 2583)
- **Archivo:** `video_generator/cli.py`

### Fix 2 — Generator "No hay audios para BOF 32"
`_select_audio()` filtraba `WHERE bof_id = ? AND id IN (audio_ids)` pero audios nuevos registrados via dashboard no tienen bof_id asignado (es nullable).
- **Fix:** Cambiado a `WHERE producto_id = ? AND id IN (audio_ids)`
- **Archivo:** `video_generator/generator.py`

### Fix 3 — Cache .pyc
Despues de editar generator.py, el debug log no aparecia. Python seguia usando el .pyc compilado anterior.
- **Fix:** `del __pycache__/generator.cpython-*.pyc`

### Fix 4 — FFmpeg "Error opening input file ."
Material registrado desde el dashboard (boton "+ Añadir") tenia `filepath = ''` vacio. FFmpeg recibia string vacio.
- **Fix:** Nuevo metodo `_resolve_filepath()` que construye ruta completa: `RECURSOS_BASE/producto/{hooks|brolls|audios}/filename`
- Aplicado a hooks y brolls en `_load_material()`, y a audios en `_select_audio()`
- **Archivo:** `video_generator/generator.py`

### Resultado
Video generado correctamente. **QUA-70 cerrado.**

## Parte 12: QUA-193 — Programador web

### Contexto
Programar calendario requeria usar el CLI (programador.py) desde la terminal. Sara queria poder hacerlo desde el dashboard web, coexistiendo con el CLI.

### Implementacion

**Nuevo archivo:** `autotok-api/api/programar.py` (~773 lineas)

**Algoritmo:** Replica completa de programador.py adaptada a Vercel serverless:
- `_load_cuenta_config()`: carga config de cuenta desde cuentas_config en Turso
- `_get_available_videos()`: videos con estado Generado para la cuenta
- `_categorize_videos()`: clasifica por lifecycle (top_seller/validated/testing)
- `_build_queue()`: cola priorizada por lifecycle con distribucion porcentual
- `_generate_time_slots()`: genera horarios equiespaciados
- `_schedule()`: asigna videos a slots respetando restricciones
- `_execute_schedule()`: UPDATE masivo en Turso

**UI web:**
- Formulario: cuenta, dias, fecha inicio, videos/dia, filtro producto
- Simulacion dry-run con preview tipo calendario + estadisticas
- Boton "Programar" para ejecutar (solo habilitado si hay videos)

**POST actions:** `load_config`, `dry_run`, `execute`

**Auth:** PIN hash identico a estado.py (`sha256("autotok-dash-{PIN}")[:16]`)

**Nav:** Link "Programar" añadido a todas las paginas (estado, stats, formatos)

### Bugs corregidos durante testing
1. **PIN hash mismatch** → formula no coincidia con estado.py (commit `57ffd6a`)
2. **Slot distribution all-zero** → con 1 slot y 40/40/20%, round() daba 0 a todas. Fix: si solo una categoria tiene videos, asignarle todos los slots (commit `b895758`)
3. **Nav link missing** → solo estaba en estado.py, faltaba en stats.py y formatos.py (commit `089ce0a`)
4. **Boton Programar disabled** → nunca se quitaba `disabled` despues de simulacion (commit `a85b671`)

**Archivo:** `autotok-api/api/programar.py` (NUEVO)
