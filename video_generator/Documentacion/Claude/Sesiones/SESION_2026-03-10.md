## Parte 2: QUA-183 — Import histórico TikTok Studio (lotopdevicky)

### Problema inicial: DOM scraping insuficiente

El script `scroll_capture` solo capturaba ~190 videos de ~745 porque TikTok Studio usa virtual rendering (solo 20-30 elementos DOM visibles a la vez, destruye el resto).

### Solución: Network interceptor

Creado `intercept_tiktok.js` — monkey-patches `fetch` y `XMLHttpRequest` para capturar las respuestas de la API interna de TikTok Studio (`item_list/v1`, 10 videos por página).

**Resultado:** 265 videos capturados (vs 190 del DOM), rango Oct 2025 → Mar 2026, 337K views totales.

**Archivo:** `Documentacion/Referencia/csv tiktokstudio/intercept_tiktok.js`

### Estrategia de matching (definida por Sara)

1. **Ya tiene post_id** → no tocar
2. **Match por SEO text + fecha (±2 días)** → asignar post_id automáticamente
   - Primera línea de `variantes_overlay_seo.seo_text` = título del video en TikTok
3. **Anteriores al 7 feb** → pre-kevin (videos manuales, sin entrada en BD)
4. **Post 7-feb sin match SEO** → lista de revisión manual (NO crear entradas nuevas, porque se pierde la info de hook/audio/producto)

### Implementación: import_studio_html.py v5

Cambios principales:
- `parse_json_output()`: soporte formato JSON del interceptor
- `_seo_first_line()`: extrae primera línea del SEO text para matching
- `match_video()`: nueva estrategia SEO + fecha ±2 días
- `--prekevin-date`: parámetro para separar pre-kevin de post-sistema
- Genera `revision_manual_{cuenta}.txt` con videos no matcheados

| Ya tenían post_id | 156 |
| Nuevos por SEO match | 97 (primera ejecución) |

**Nota:** Primera ejecución crasheó en INSERT pre-kevin (NOT NULL en producto_id). Los 97 SEO matches ya estaban committed (Turso autocommit). Se corrigió para que pre-kevin solo vaya a review file.

### Archivos generados

- `revision_manual_lotopdevicky.txt` — 109 videos para revisión manual (7 pre-sistema + 102 post-sistema sin match SEO)

### Contexto y decisiones estratégicas

Sara definió un cambio de enfoque: **llevar todo lo posible al panel web** para minimizar errores humanos y facilitar la gestión diaria. Las áreas clave:

### Decisiones de Sara (correcciones al análisis inicial)

**Renaming:**
- BOF → **Formato** (UI y documentación)
- deal_math → **deal** (UI)
- BD mantiene nombres legacy (`producto_bofs`, `deal_math`) para compatibilidad con video_generator

**Campos:**
- Añadir `precio` a formatos (opcional)
- Eliminar `caracteristicas` del flujo (campo existe en BD pero no se usa)
- Añadir `marca` a productos

**Multi-link:**
- Cada URL = cada formato. No se necesita tabla separada `producto_links`
- Un producto puede tener múltiples formatos con distintas URLs

**Auto-generación:**
- `bof_generator.py` se descarta como módulo (no funciona bien)
- SE MANTIENE: generación de overlay (6 variantes), SEO text (12 templates), hashtags (max 7)
- SE DESCARTA: templates de hooks (100), generación de guion
- Resultados auto-generados son editables en el panel
- No reducir opciones generadas (se necesitan más combinaciones para videos)

**Guion audio:**
- Input manual de texto (no auto-generado)

**Audios:**
- Solo voz humana (no TTS)
- Dropdown de audios registrados en BD para asociar a formatos

**Materiales:**

### Arquitectura del endpoint /api/formatos

**API (`formatos.py`):**
- GET `/api/formatos?producto_id=X` — Lista formatos con variantes y audios asociados
- POST `/api/formatos` — Crear formato + auto-generar variantes overlay/SEO/hashtags
- PUT `/api/formatos?id=X` — Editar formato (deal, url, precio, guion, hashtags) y variantes individuales
- DELETE `/api/formatos?id=X` — Desactivar (activo=0)

**Auto-generación (portada de bof_generator.py):**
- `generar_variaciones_overlay(marca, producto, deal, n=6)` — 6 combos línea1+línea2 con variantes de deal y urgencias
- `generar_seo_text(marca, producto, deal, variacion_num)` — 12 templates de SEO
- `generar_hashtags(marca, producto)` — max 7 hashtags

**UI integrada en `/productos`:**
- Al hacer clic en un producto se expande sección de formatos
- Formulario para crear formato: deal, URL, precio (opt), guion (textarea)
- Auto-genera variantes al crear → muestra en tabla editable
