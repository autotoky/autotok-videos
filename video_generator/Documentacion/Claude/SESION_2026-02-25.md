### 2. Análisis de automatización TikTok

**Contexto:** 2→4 cuentas × 25 vídeos/día = 100 vídeos/día. Programación manual = 2h/cuenta = 8h/día.

**5 opciones analizadas:**

| Opción | Resultado | Motivo |
|--------|-----------|--------|
| A: API oficial | ❌ | Límite ~15 posts/día |
| B: SaaS (Late, PostFast) | ❌ | Misma API, mismo límite |
| C: Playwright/Selenium | ✅✅ ELEGIDA | Riesgo bajo con mitigaciones |
| D: ADB (Android) | ❌ | Complejo para Carol/Vicky |
| E: Híbrida | ✅ Fase futura | API + navegador |

**Correcciones del usuario:**
- Bulk upload de TikTok Studio NO disponible en todas las cuentas → descartada
- Tiempo real por cuenta = 2h (no 45-60 min como estimaba inicialmente)
- Producto se selecciona del escaparate TikTok Shop, no se pega URL
- titulo_publicacion = refuerzo promo ("Últimas unidades"), no nombre producto
- **Aclaración clave de riesgo:** El riesgo NO es usar navegador (ya usan Chrome + TikTok Studio a diario), sino que TikTok detecte que un SCRIPT controla el navegador. Con mitigaciones (stealth, perfil real, delays aleatorios), es prácticamente indistinguible del uso manual.

**Documento generado:** `analisis_automatizacion_tiktok.docx` (v2, con decisión y motivos)

### 3. Decisión: Opción C (Playwright + stealth)

**5 motivos principales:**
1. Replica exactamente el flujo manual actual
2. Usa el mismo Chrome con las mismas cookies
3. Mitigaciones anti-detección hacen el riesgo muy bajo
4. ADB descartada por complejidad para operadores
5. Desarrollo y mantenimiento más rápido que ADB

### 4. Prototipo Publisher v1 — Implementación

**`tiktok_publisher.py`** — Módulo principal (500+ líneas):
- Clase `TikTokPublisher` con Playwright + stealth
- `get_videos_para_publicar(cuenta, fecha)` — Query BD con todos los metadatos
- `marcar_video_publicado(video_id)` — Actualiza estado en BD
- Anti-detección: `--disable-blink-features=AutomationControlled`, navigator.webdriver override, ignore_default_args=['--enable-automation']
- Perfil real de Chrome con `launch_persistent_context`
- Delays aleatorios: entre videos (15-30s), entre acciones (1.5-4s), typing humano (30-80ms/char)
- Flujo por video: upload → descripción (SEO+hashtags) → título → [escaparate TODO] → programar fecha/hora → confirmar
- Dry run mode para simular sin tocar TikTok
- Logging completo a archivo + screenshots en caso de error
- CLI independiente: `python tiktok_publisher.py --cuenta X --fecha Y`
- Función `listar_pendientes()` para ver videos por publicar

**`config_publisher.json`** — Configuración por cuenta:
- Ruta al perfil de Chrome de cada operadora
- Nombre del profile de Chrome (Default, Profile 1, etc.)
- Título por defecto para publicaciones
- Ruta al ejecutable de Chrome

**`scripts/setup_publisher.py`** — Instalador:
- Verifica Python 3.8+
- Busca Chrome instalado
- Instala Playwright + descarga Chromium
- Verifica config_publisher.json

**`cli.py`** — 2 opciones nuevas:
- Opción 19: 📋 Ver videos pendientes de publicar
- Opción 20: 🚀 Publicar videos en TikTok Studio
